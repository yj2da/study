## 1. 데이터 엔지니어링이란?

**데이터 엔지니어링**은 **분산 시스템을 활용해서 데이터를 순차적으로 가공해 나가는 구조**를 다루는 일입니다.

### 핵심 개념

- **데이터 파이프라인 구축**: 데이터가 수집 → 저장 → 처리 → 분석 → 활용되는 전체 흐름을 설계
- **분산 시스템 활용**: 여러 서버를 조합해 확장성 있는 데이터 처리 구조 구현
- **자동화**: 반복적인 데이터 처리 작업을 자동화하여 효율성 극대화

### 데이터의 여정

```mermaid
flowchart LR
    A[수집] --> B[저장]
    B --> C[처리]
    C --> D[분석]
    D --> E[활용]
```

## 2. 왜 데이터 엔지니어링이 중요한가?

### 📊 데이터 중심 의사결정의 시대

- 모든 기업이 **데이터 기반 의사결정**을 추구
- 데이터 없이는 비즈니스 인사이트를 얻을 수 없음

### 🤖 AI/ML의 기반

- AI/ML 모델은 **좋은 데이터**가 있어야 학습 가능
- 데이터 엔지니어가 **양질의 데이터를 안정적으로 공급**

### 🌐 실제 사례

- **넷플릭스**: 추천 시스템을 위한 대규모 데이터 파이프라인
- **우버**: 실시간 위치 데이터 처리 및 분석
- **네이버**: 검색 품질 향상을 위한 데이터 플랫폼
  - 참고: https://naver-career.gitbook.io/kr/service/search/ai-and-data-platform

## 3. 데이터 엔지니어 vs 다른 역할

데이터 관련 직군은 크게 세 가지로 나뉩니다:

- **데이터 엔지니어**: 시스템 구축·운영·자동화
- **데이터 사이언티스트**: ML 모델 개발
- **데이터 분석가**: 인사이트 도출·보고서

자세한 역할 구분은 [데이터 엔지니어링 개념 문서](../../부경대학교/외부용/Day1/02_데이터엔지니어링_개념.md#데이터-사이언티스트-vs-데이터-엔지니어-vs-데이터-분석가)를 참고하세요.

## 4. 데이터 파이프라인의 여정

### 1️⃣ 데이터 수집

**벌크(Bulk)형**
- 이미 존재하는 데이터를 정기적으로 수집
- 예: 매일 밤 DB에서 데이터 추출

**스트리밍(Streaming)형**
- 실시간으로 생성되는 데이터를 끊임없이 전송
- 예: 모바일 앱 이벤트 로그, 센서 데이터

### 2️⃣ 데이터 저장

- **데이터 레이크**: 원시 데이터를 있는 그대로 저장 (예: S3, HDFS)
- **데이터 웨어하우스**: 구조화된 데이터를 장기 보존 (예: Snowflake, BigQuery)
- **데이터 마트**: 특정 목적을 위해 가공된 데이터 저장

### 3️⃣ 데이터 처리

**배치 처리**
- 일정 시간마다 모인 데이터를 한 번에 처리
- 예: 매일 밤 집계 작업

**스트림 처리**
- 실시간으로 들어오는 데이터를 즉시 처리
- 예: 실시간 대시보드, 이상 탐지

**ETL (Extract-Transform-Load)**
- 데이터 추출 → 변환 → 적재 과정
- 데이터 품질 관리 및 표준화

### 4️⃣ 분석 및 활용

- SQL 쿼리를 통한 데이터 분석
- 대시보드 및 시각화
- ML 모델 학습용 데이터 제공

## 5. 실무에서 사용하는 기술 스택

![데이터 엔지니어링 기술 스택](../img/data_engineering.png)

### 📦 분산 데이터 저장

- **HBase**: Hadoop 기반 NoSQL 데이터베이스
- **Kafka**: 분산 스트리밍 플랫폼 (메시지 큐)
- **MongoDB**: 도큐먼트 기반 NoSQL
- **MySQL**: 관계형 데이터베이스
- **Redis**: 인메모리 데이터 저장소
- **Elasticsearch**: 검색 및 분석 엔진

### ☁️ 클라우드 & 분산 시스템

- **Kubernetes (K8s)**: 컨테이너 오케스트레이션 및 **서버 운영**
- **Spring Cloud**: 마이크로서비스 아키텍처 프레임워크
- **Yarn**: Hadoop 리소스 관리자
- **Docker**: 컨테이너화 플랫폼
- **Network Server**: 네트워크 서버 구축 및 운영
- **분산 환경 보안**: 분산 시스템 보안 관리

### 💻 개발 언어 & 프레임워크

**백엔드 언어**
- **Python**: 데이터 처리의 표준 언어
- **Java**: 엔터프라이즈 환경의 핵심
- **Scala**: Spark와 함께 사용
- **Golang**: 고성능 서비스 개발
- **C++, C**: 시스템 레벨 개발
- **Kotlin**: 모던 JVM 언어

**프론트엔드 & 웹**
- **JavaScript / TypeScript**: 웹 개발
- **Node.js**: 서버사이드 JavaScript
- **React**: UI 라이브러리
- **Spring**: Java 기반 웹 프레임워크

**기타 라이브러리**
- **Hugging Face Transformer Library**: NLP/ML 모델
- **Freemarker**: 템플릿 엔진

### 🔧 빅데이터 처리 & 분석

**Hadoop 생태계**
- **Hadoop**: 분산 파일 시스템 및 처리 프레임워크 (Eco System 포함)
- **Spark**: 대규모 데이터 처리 엔진
- **Hive**: SQL 기반 데이터 웨어하우스
- **Trino (Presto)**: 분산 SQL 쿼리 엔진
- **Hudi**: 데이터 레이크 관리 프레임워크

**스트림 & 실시간 처리**
- **Flink**: 실시간 스트림 처리
- **Kafka**: 스트리밍 데이터 수집

**워크플로 & ETL**
- **Airflow**: 워크플로 스케줄링 및 관리
- **ETL**: 데이터 추출·변환·적재 파이프라인

**로그 & 모니터링**
- **ELK Stack**: Elasticsearch, Logstash, Kibana (운영 포함)
- **Logstash**: 로그 수집 및 처리

### 🛠️ 기타 도구

- **Linux**: 서버 운영 환경
- **Jenkins**: CI/CD 자동화
- **Git**: 버전 관리 및 **코드 리뷰**
- **Webpack**: 모듈 번들러
- **Headless Browser**: 웹 스크래핑 및 테스트

---

### 💡 실무 기술 스택의 특징

데이터 엔지니어링은 **매우 다양한 기술 스택**을 다룹니다:

- **데이터 저장**: 6가지 이상의 다양한 저장소
- **언어**: 10개 이상의 프로그래밍 언어
- **빅데이터 처리**: Hadoop 생태계 전반
- **인프라**: K8s, Docker 등 현대적인 클라우드 환경
- **DevOps**: CI/CD, 모니터링, 보안까지

한 분야에 국한되지 않고, **전체 데이터 생태계**를 경험할 수 있는 것이 데이터 엔지니어의 큰 장점입니다!

## 6. 데이터 엔지니어의 장점

### 🚀 다양한 기술 스택 경험

데이터 엔지니어링은 **세부적으로 나누면 분야가 매우 다양**합니다:

- **데이터 수집**: Kafka, Logstash, Fluentd
- **데이터 저장**: 다양한 DB와 스토리지 시스템
- **데이터 처리**: Spark, Flink 등 분산 처리 프레임워크
- **인프라**: K8s, Docker, 클라우드 서비스
- **자동화**: Airflow, CI/CD 파이프라인

한 분야만 파는 것이 아니라, **전체 데이터 생태계**를 경험할 수 있습니다!

### 🛠️ DevOps와 서비스 운영 경험

데이터 엔지니어는 결국 **서비스를 다루게** 됩니다:

- **Kubernetes (K8s)**: 컨테이너 오케스트레이션 환경 구축 및 운영
- **CI/CD**: ArgoCD, Jenkins 등을 활용한 자동 배포
- **모니터링**: 서비스 안정성을 위한 모니터링 시스템 구축
- **자동화**: 반복 작업을 자동화하는 시스템 개발

단순히 데이터만 다루는 것이 아니라, **전체 서비스 인프라를 이해**하고 운영하는 능력을 기를 수 있습니다.

### 📈 성장 가능성

- 아직 **작은 분야**지만, 그만큼 **발전할 게 무궁무진**
- **데이터 파이프라인**은 갈수록 중요해지는 추세
- AI/ML의 확산으로 데이터 엔지니어의 수요 증가

### ⚖️ 워라밸

- 서비스 조직에 비해 **배포 민감도가 낮음**
- 내부 직원을 상대하기 때문에 **일정 압박이 덜함**
- **안정적인 업무 환경**

## 7. 데이터 엔지니어링의 미래

### 🔮 트렌드

**1. 실시간 처리의 중요성 증가**
- 배치에서 스트림으로의 전환
- Lambda → Kappa → Kappa+ 아키텍처 진화

**2. AI/ML과의 통합**
- MLOps: ML 모델 배포 및 관리
- Feature Store: ML 피처 관리 시스템
- 데이터 품질이 모델 성능을 좌우

**3. 클라우드 네이티브 아키텍처**
- 서버리스 데이터 처리 (AWS Lambda, Google Cloud Functions)
- 관리형 서비스 활용 증가 (Snowflake, Databricks)

**4. 데이터 거버넌스와 보안**
- 개인정보보호법 강화 (GDPR, CCPA)
- 데이터 리니지(Lineage) 추적
- 데이터 품질 관리 자동화

**5. 오픈소스 생태계 성장**
- Apache 프로젝트들의 지속적인 발전
- 커뮤니티 기반 혁신

### 💡 기회

- **작은 분야**이지만 **발전 가능성 무한**
- **다양한 기술 스택** 경험 가능
- **DevOps와 인프라** 경험으로 커리어 확장 가능
- **안정적인 워라밸**과 **성장 기회** 모두 잡을 수 있는 분야
