### 자주 나오는 질문 (FAQ)

**Q: Docker와 Podman 중 뭘 쓰면 되나요?**  
A: 둘 다 Docker 호환 명령을 씁니다. **Docker Desktop**은 기업용 라이선스 제한이 있어서, 회사 PC에서는 **Podman Desktop**을 쓰는 경우가 많습니다. 실습은 Podman 기준으로 했고, Docker 사용자는 `podman` → `docker`로 치환하면 됩니다.

**Q: Linux를 꼭 배워야 하나요?**  
A: 서버·클라우드·컨테이너 대부분이 Linux 기반이라, **로그 확인·설정 파일·명령어**를 다루는 일이 많습니다. GUI 없이 터미널만 있는 환경도 있어서, 기본 명령어(pwd, ls, cd, grep, tail 등)는 익혀 두는 게 좋습니다.

**Q: 데이터 엔지니어와 데이터 사이언티스트 차이가 뭔가요?**  
A: **엔지니어**는 파이프라인·저장·처리 **시스템을 설계·구축·운영**하고, **사이언티스트**는 그 위에서 **통계·ML 모델을 개발**하는 쪽에 가깝습니다. 02에서 역할·필요 기술 표로 정리해 두었습니다.

**Q: Multi-stage 빌드를 왜 쓰나요?**  
A: 빌드할 때만 쓰는 도구(컴파일러, 소스 등)를 **최종 이미지에 넣지 않고**, 실행에 필요한 것만 넣어서 **이미지 크기를 줄이기 위해** 씁니다. 배포·풀(pull)이 가벼워지고, 보안 면에서도 유리합니다.

**Q: Day2 전에 뭘 해두면 좋을까요?**  
A: 오늘 쓴 **Docker/Podman 환경이 정상 동작하는지** 한 번 더 확인해 두면 좋습니다. (선택) Day2에서 다룰 **HDFS**가 무엇인지, 분산 저장이 왜 필요한지 미리 검색해 보면 이해에 도움이 됩니다.
