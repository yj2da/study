# 04. Linux 기본 명령어 소개

## 왜 Linux·Docker 실습을 하나요? (사전 설명)

이 강의에서는 이어서 **Linux 명령어 실습**과 **Docker로 실습 환경 구성**을 합니다. 
갑자기 나온 느낌이 들 수 있어서, 왜 이걸 하는지 짚고 넘어갑니다.

- **회사에 가면** 오픈소스를 가져다 쓰고, API 서버를 운영하고, 요즘은 CI/CD도 잘 갖춰져 있고, AWS 같은 클라우드에서 많이 돌립니다. 그럼에도 **Linux·쉘·컨테이너 같은 기본**은 여전히 알아야 합니다.
- **LLM·에이전트**에게 물어보면 명령어나 설정을 잘 알려주긴 합니다. 다만 "**이게 뭔지 알고 쓰는 것**"과 "**모르고 쓰는 것**"은 다릅니다. 이해도가 다르면 디버깅·설정 변경·원인 추적이 달라지고, 결국 **개발 속도**도 달라집니다.
- **문제가 생겼을 때** 로그 확인, 프로세스·포트 확인, 파일 권한·경로 점검처럼 **명령어를 잘 쓸 줄 아는 것**이 원인 파악의 기본입니다. 이번 시간에는 그 기본을 조금이라도 익혀 두자는 목적입니다.

아래에서는 그중 **Linux 기본 명령어**만 먼저 다룹니다. Docker 실습은 다음 파일(06)에서 이어서 진행합니다.

---

## 왜 Linux인가?

- **서버·클라우드·컨테이너** 대부분이 Linux 기반입니다. AWS EC2, Docker 컨테이너, 빅데이터 클러스터(Hadoop, Spark 등)도 Linux 위에서 동작합니다.
- **데이터 파이프라인**을 만들 때 로그·설정 파일·스크립트를 다루는 환경이 대부분 터미널(Linux/Unix 쉘)입니다.
- **macOS도 Unix 기반**이라 동일한 Linux 명령어를 그대로 사용할 수 있습니다. 이것이 **개발자들이 Mac을 많이 선택하는 이유 중 하나**입니다.
- Windows만 써왔어도 **WSL(Windows Subsystem for Linux)** 이나 **Git Bash**로 비슷한 명령어를 쓸 수 있으니, 여기서 익힌 것만으로도 도움이 됩니다.

## 필수 명령어 소개

### 디렉토리 탐색

#### `pwd`
- **Print Working Directory** — 지금 있는 디렉토리 경로를 출력합니다.
- 예: `pwd` → `/home/user/project`

#### `ls`
- 디렉토리 **내용(파일·폴더 목록)** 을 봅니다.
- `ls -l` : 자세히(권한, 크기, 날짜 등), `ls -a` : 숨김 파일 포함

#### `cd`
- **Change Directory** — 디렉토리를 이동합니다.
- `cd 폴더명`, `cd ..` (한 단계 위), `cd ~` (홈 디렉토리), `cd -` (이전 디렉토리)

---

### 파일 조작

#### `cp`
- **복사**. `cp 원본 대상` 또는 `cp 원본1 원본2 ... 폴더/` (여러 개를 한 폴더로).
- 디렉토리 통째로: `cp -r 디렉토리 대상`

#### `mv`
- **이동** 또는 **이름 변경**. `mv 원본 대상` (같은 경로면 이름만 바뀜).

#### `rm`
- **삭제**. `rm 파일` (복구 안 됨). 디렉토리: `rm -r 폴더`. 확인 없이: `rm -f` (주의).

#### `mkdir`
- **디렉토리 만들기**. `mkdir 이름`, 여러 단계 한 번에: `mkdir -p a/b/c`

---

### 파일 내용 확인

#### `cat`
- 파일 **전체 내용**을 터미널에 출력. 작은 파일·로그 확인에 자주 씀.  
  예: `cat config.yaml`

#### `head`
- **앞부분**만 봄. 기본 10줄. `head -n 5 파일` → 처음 5줄.

#### `tail`
- **뒷부분**만 봄. 로그를 실시간으로 보려면 `tail -f 로그파일` (Ctrl+C로 종료).

#### `less`
- **한 화면씩** 스크롤하며 보기. 큰 파일에 유리. `q`로 종료, `/검색어`로 검색.

#### `vi` / `vim`
- **파일 편집기**. `vi 파일명` 또는 `vim 파일명`으로 파일을 열고 수정할 수 있습니다.
- 다만 **특정 환경(특히 최소화된 Docker 이미지)에서는 설치가 안 되어 있을 수 있습니다**. 
- Docker 환경에서 서버를 운영할 때 이미지 크기를 줄이기 위해 최소한의 패키지만 포함하는 경우가 많기 때문입니다.
- 이런 환경에서는 `cat`, `head`, `tail` 등으로 파일 내용만 확인하거나, 필요시 `echo` 또는 호스트에서 편집 후 복사하는 방식을 사용합니다.

---

### 검색

#### `grep`
- **텍스트에서 패턴 검색**. `grep "에러" 로그파일`, `grep -r "keyword" ./` (하위 디렉토리까지).
- `-i` : 대소문자 구분 없음, `-n` : 줄 번호 표시.

#### `find`
- **파일·디렉토리 찾기**. `find . -name "*.log"`, `find . -type d -name src`.
- `-type f`(파일), `-type d`(디렉토리), `-mmin -60`(60분 이내 수정) 등 조합 가능.

## 권한 관리 기초

### `chmod`
- **권한 변경**. 읽기(r)=4, 쓰기(w)=2, 실행(x)=1. 숫자 합으로 지정.  
  예: `chmod 755 script.sh` (소유자 rwx, 그룹·기타 rx), `chmod +x script.sh` (실행 권한 추가).

### `chown`
- **소유자·그룹 변경**. `chown 사용자:그룹 파일`. Docker·서비스 설정할 때 권한 문제 해결할 때 자주 사용.

## 핵심 개념 정리

- **디렉토리 탐색**: `pwd`(현재 경로), `ls`(목록), `cd`(이동). 실무에서 로그·설정 파일 위치 찾을 때 기본.
- **파일 조작**: `cp`(복사), `mv`(이동·이름변경), `rm`(삭제, 주의), `mkdir`(디렉토리 생성).
- **파일 내용 확인**: `cat`(전체), `head`/`tail`(앞·뒤 N줄), `tail -f`(실시간 로그), `less`(큰 파일 스크롤·검색).
- **검색**: `grep`(텍스트 패턴), `find`(파일·디렉토리 찾기). 원인 파악할 때 자주 씀.
- **권한**: `chmod`(권한 변경), `chown`(소유자·그룹). Docker·서비스 배포 시 권한 이슈 해결에 필요.
