# Kubernetes CI/CD 도구 간단 정리

> Kubernetes 환경에서 자주 쓰이는 CI/CD·배포 도구들을 역할별로 요약한 문서입니다.

---

## 1. 개요

| 구분 | 역할 | 대표 도구 |
|------|------|-----------|
| **CI** | 빌드·테스트·이미지 푸시 | Jenkins, GitHub Actions, GitLab CI, Tekton |
| **CD (GitOps/배포)** | 클러스터에 배포·동기화 | Argo CD, Flux |
| **CD (멀티클라우드·파이프라인)** | 복잡한 파이프라인·멀티클라우드 | Spinnaker |
| **개발 워크플로** | 로컬/개발 환경에서 빌드→배포 반복 | Skaffold, Tilt, Garden |

- **Argo CD / Flux**: Git을 소스 오브 트루스로 두고, k8s 매니페스트(또는 Helm/Kustomize)를 클러스터에 **선언적으로 동기화**하는 **GitOps CD** 도구.
- **Spinnaker**: 여러 클라우드·복잡한 배포 전략(카나리, 롤백 등)을 **파이프라인**으로 다루는 **CD 플랫폼**.
- **Skaffold**: 개발자가 코드 수정 후 **빌드 → 푸시 → k8s 배포**를 한 번에 돌리는 **개발용 CLI** (프로덕션 CD가 아님).

---

## 2. Argo CD

- **역할**: Kubernetes 전용 **GitOps 기반 CD**. Git 저장소의 매니페스트(Helm, Kustomize, plain YAML)와 클러스터 상태를 **지속적으로 비교·동기화**.
- **특징**
  - Git을 **단일 소스 오브 트루스**로 사용. 선언적(declarative).
  - 애플리케이션 단위로 상태·헬스 체크, 자동/수동 Sync.
  - **멀티 클러스터** 배포 지원.
  - CNCF 졸업 프로젝트, 커뮤니티·문서 활발.
- **적합한 경우**: k8s 중심, GitOps로 단순하고 명확한 CD 파이프라인을 원할 때.

---

## 3. Spinnaker

- **역할**: **멀티클라우드 CD 플랫폼**. AWS, GCP, Azure, k8s 등 여러 타겟에 대한 배포 파이프라인·전략을 하나의 플랫폼에서 관리.
- **특징**
  - **마이크로서비스 아키텍처**로 구성된 대형 CD 시스템.
  - **카나리 분석**, 트래픽 분할, 롤백 등 고급 배포 전략.
  - 파이프라인 단계(빌드·테스트·스테이징·프로덕션)를 시각적으로 정의.
  - 설치·운영 복잡도와 리소스 사용이 큼.
- **적합한 경우**: 여러 클라우드·복잡한 배포 전략·대규모 조직이 하나의 CD 플랫폼을 쓸 때.

---

## 4. Skaffold

- **역할**: **개발자 로컬/개발 환경**에서 “코드 수정 → 빌드 → 이미지 푸시 → k8s 배포”를 **한 번에 반복**하기 위한 CLI.
- **특징**
  - `skaffold dev` 실행 시 파일 변경 감지 → 자동 빌드·배포(dev loop).
  - Dockerfile/Jib/Buildpacks 등으로 빌드, k8s 매니페스트(kubectl/helm/kustomize)로 배포.
  - **프로덕션 CD가 아니라** 빠른 개발 피드백용. CI/CD 파이프라인에서 `skaffold build` / `skaffold deploy`를 단계로 넣을 수는 있음.
- **적합한 경우**: k8s에 올려서 바로 확인하면서 개발할 때, Tilt·Garden과 같은 “개발용 워크플로” 도구를 찾을 때.

---

## 5. 그 외 참고

| 도구 | 한 줄 요약 |
|------|------------|
| **Flux** | GitOps CD. Argo CD와 유사하게 Git ↔ 클러스터 동기화. CLI·컨트롤러 중심, 가벼운 구성 선호 시. |
| **Tekton** | k8s 리소스(CRD) 기반 **CI/CD 파이프라인 엔진**. 파이프라인을 k8s 네이티브로 정의할 때. |
| **Jenkins X** | Jenkins 기반 k8s용 CI/CD. GitOps·프리뷰 환경 등 포함. (커뮤니티/운영 모델 변화 있음.) |
| **Tilt** | 로컬 k8s 개발용. Skaffold과 비슷하게 “코드 변경 → 빌드 → 배포” 자동화. |

---

## 6. 선택 시 참고

- **k8s만 쓰고 GitOps로 단순 CD** → **Argo CD** 또는 **Flux**.
- **멀티클라우드·복잡한 파이프라인·카나리 등** → **Spinnaker** 검토.
- **로컬/개발에서 빌드·배포 반복** → **Skaffold** 또는 **Tilt**.
- **파이프라인 자체를 k8s 리소스로 관리** → **Tekton** 검토.
